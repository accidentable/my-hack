# ComplianceLens — 멀티모달·다국어 금융 콘텐츠 사전심의 AI Agent

> JB금융그룹 Fin:AI Challenge — **지정주제 2. Compliance AI (준법자문가 AI Agent)** 기획 정리

---

## 0. 한 줄 정의

카드뉴스·릴스·포스터 등 **멀티모달·다국어** 금융 콘텐츠를 게시 전에 자동 사전심의하여,
위반 소지·근거·수정안을 정리하고 **준법관리자는 검토·승인만** 하도록 전환하는 AI Agent.

---

## 1. 왜 이 주제인가 (대회 맥락)

- **대회**: JB금융그룹 × 데이콘 공동 주최, 바이브코딩 해커톤 플랫폼 DAKER에서 진행. 청년(19~34세) 대상, 직장인 불가. 총상금 1억원. 이번이 사실상 첫 회차(신규 플랫폼).
- **JB가 좋아하는 답안**: 직전 내부 해커톤(2026.2) 대상작이 "AI 기반 여신심사·분석 솔루션"이었고, 참가자평이 *"기술 자랑보다 사업 구조를 바꾸는 것"* 이었음 → **사업 임팩트 + 실제 동작하는 데모**가 핵심.
- **지정주제 2를 택한 결정적 이유**: 직접 겪은 병목이 곧 주제다. 투자 콘텐츠(카드뉴스·릴스) 제작 시 준법심의가 한 달 넘게 걸려, 유행이 다 지난 뒤 게시되는 일이 반복됨. 공고가 적은 문제와 토씨 하나 안 틀리게 일치.

## 2. 문제 정의 (Problem)

### 2.1 발생하는 문제
- 금융 대고객 콘텐츠는 준법관리자가 **전량 수작업**으로 심의 → 수 주~한 달 소요, 배포 시점 지연.
- 외국어(다국어) 콘텐츠도 동일 인력·동일 방식 → **언어·채널이 늘수록 심의 리소스가 선형 증가**.
- 담당자·시점별 **품질 편차와 휴먼에러**, 일관성·추적성 부족.

### 2.2 실증 Pain Point
- 실제 금융 콘텐츠 크리에이터로서, 카드뉴스·릴스 심의에 한 달 이상 소요되어 **콘텐츠 적시성을 상실**한 경험. (발표 오프닝 서사로 사용)

### 2.3 해결 대상
- 금융사 **준법감시(Compliance) 부서** + 콘텐츠/마케팅 제작자.

### 2.4 중요성 / JB 관점
- JB는 **외국인 금융 1위**(전북은행 외국인 신용대출 시장 72% 점유). 외국인 대상 콘텐츠는 담당자가 해당 언어를 못 읽어 심의가 더 느리거나 불가 → **다국어 심의 병목이 JB에 특히 치명적**. 주제가 다국어를 명시한 이유와 정확히 맞물림.

### 2.5 기대 변화 (KPI)
- 1차 심의 리드타임: **주 단위 → 분 단위**.
- 건당 검토시간 절감, **다국어 심의 커버리지** 확보.
- 외국인 콘텐츠 미심의·불완전판매 리스크 최소화.

## 3. 핵심 통찰 — 이 문제가 진짜로 필요로 하는 것

> 필요한 건 "기능"이 아니라 **신뢰할 수 있는 판정의 근거(설명가능성)** 다.

- 심의가 느린 진짜 이유는 **판정에 책임이 따르기 때문**(틀리면 제재·소송). 그래서 함부로 통과시킬 수 없음.
- AI가 "통과"라고만 하면 아무도 못 믿어 사람이 처음부터 다시 봄 → 병목 그대로.
- AI가 **"어느 규정 위반 / 왜 / 근거 / 수정안"**까지 제시하면 → 준법관리자는 **검토·승인만** → 병목 해소.
- 즉 멀티모달·다국어는 *입력을 넓히는 기능*이고, **근거 제시는 그 입력을 신뢰 가능한 결정으로 바꾸는 심장**.

**우선순위**
1. (심장) 근거 기반 판정 + 사람 검토 인터페이스
2. (차별화) 멀티모달 입력 — 카드뉴스·릴스에도 작동
3. (JB 정합성) 다국어 — 베트남어 등 원문 심의

비유: 이 에이전트는 "심의를 대신하는 기계"가 아니라 **"준법관리자가 1초 만에 결재할 수 있게 서류를 완벽히 정리해주는 보좌관"**. 결정권은 사람에게.

## 4. 솔루션 구조 — 6단계 닫힌 루프

```
① 수집(Ingest) → ② 검색(Retrieve) → ③ 판단(Assess) → ④ 생성(Generate) → ⑤ 검증(Verify) → ⑥ 사람 검토(Human Review)
                                                                                              └──(본선) 검토 의견 학습 → 판정 개선 ──┘
```

| 단계 | 이름 | 하는 일 |
|---|---|---|
| ① 수집 | Ingest | 이미지(OCR)·영상(프레임/자막)·음성(STT)에서 모든 요소를 추출해 **검사 가능한 '주장' 단위로 분해** |
| ② 검색 | Retrieve | 콘텐츠 유형 판별 후 해당 규제·심의기준을 RAG로 참조 |
| ③ 판단 | Assess | 주장 ↔ 규정 대조. **확정 규칙은 규칙엔진**(필수 고지 등), **모호한 표현은 LLM**(과장·오해) → 심각도·신뢰도 부여 |
| ④ 생성 | Generate | 위반·근거규정·현재표현·수정안·심각도를 담은 **심의의견서** 작성 |
| ⑤ 검증 | Verify | AI가 자기 판정을 재확인(근거 규정이 실제 그 내용인지, 환각 아닌지) → 신뢰도 표기 |
| ⑥ 사람 검토 | Human Review | 준법관리자가 항목별 **승인/반려 + 코멘트** → 에이전트가 **수정사항 정리본** 재출력 |

> 핵심: 단순 LLM 호출 파이프라인이 아니라 **자기검증(⑤) + 사람 승인(⑥)으로 닫히는 "판단·행동·검증/개선" 구조의 AI Agent**.
> (⑥의 "AI 재학습"은 본선 고도화 항목으로 이관 — 예선에서는 코멘트 취합·정리 출력까지.)

## 5. 주요 기능 (6대)

| # | 기능 | 설명 | 비고 |
|---|---|---|---|
| 1 | 멀티모달 콘텐츠 인식 | 카드뉴스 OCR·영상 프레임/자막·음성(STT)에서 주장 단위로 분해 | MVP 핵심 |
| 2 | 규제 RAG 근거 매핑 | 콘텐츠 유형별 적용 규정을 검색해 '어느 조항 위반인지' 연결 | MVP 핵심 |
| 3 | 위반 체크리스트 | 규칙기반(필수고지)+LLM(과장·오해) 판정에 심각도·신뢰도 부여 | MVP 핵심 |
| 4 | 심의의견서 자동 생성 | 위반·근거·현재표현·수정안·심각도를 담은 결재용 문서 출력 | MVP 핵심 |
| 5 | 다국어 원문 직접 심의 | 번역본이 아닌 원문(베트남어 등)을 판정하고 한국어 병기 | MVP 핵심 |
| 6 | 사람 검토 루프 | 승인·반려·코멘트 → 수정사항 정리본 재출력 (HITL) | MVP 핵심 |

### 왜 멀티모달이 필수인가
금융 콘텐츠 위반은 텍스트에만 있지 않음.
- **이미지(카드뉴스)**: 자막 속 과장 수익률, 작은 글씨로 빠진 필수 고지(원금손실 가능성), 오해 소지 그래프.
- **영상(릴스)**: 음성 멘트의 단정적 표현, 자막/음성 불일치, BGM 위 고지 누락.
- 기존엔 사람이 일일이 보고 들어야 해서 느림 → 멀티모달 LLM이 OCR+음성+프레임을 한 번에 읽으면 이 수작업이 사라짐.

## 6. 다국어 처리 원칙 (가장 중요한 설계 결정)

> **번역해서 심의하지 말고, 원문을 심의하되 사람이 검증할 수 있게 번역을 병기한다.**

- **함정**: "외국어 → 한국어 번역 → 한국어로 심의"는 위험. 번역 과정에서 원문의 표현 리스크(과장·오해 뉘앙스)가 사라지거나 없던 문제가 생김. 준법 판정은 *소비자가 실제로 보는 원문*이 기준이어야 함.
- **방식 A (코어)**: 멀티모달 LLM이 베트남어 등 **원문 그대로** 읽고, 그 맥락에서 주장 추출·판정. 규정(한국 금소법)은 한국어지만 "이 베트남어 표현이 → 원금손실 고지 누락에 해당"처럼 **원문 표현과 한국 규정을 직접 연결**.
- **방식 B (검증 보조)**: 체크리스트에 **원문/번역/근거**를 병기 → 담당자가 그 언어를 몰라도 검토 가능.
- **방식 C (차별화)**: 한국어 원본 약관·광고를 외국어로 옮길 때 **필수 고지 누락·의미 왜곡**을 검사 → 외국인 불완전판매 리스크를 정면으로 다룸.
- **데모 범위**: 베트남어 하나로 끝까지 시연, 태국어·캄보디아어는 확장 로드맵.

## 7. 데이터 & 기술

- **활용 데이터**
  - 규정 문서: 금융소비자보호법 광고규제, 금융투자협회 광고심의기준 등 *(부록에 출처·조항 명시 — 추후 확정)*
  - 모의 카드뉴스·릴스·포스터 샘플
  - 다국어(베트남어 등) 콘텐츠 샘플
- **처리 방식**
  - 멀티모달 LLM으로 이미지·자막·음성 이해 → 주장 추출
  - 임베딩 + 벡터검색(RAG)으로 규정 근거 연결
  - 규칙엔진(필수 고지 체크) + LLM 판정 결합
- **모델·도구**: 멀티모달 LLM API · OCR · STT · 벡터 DB · 규칙엔진
  - ⚠️ 외부 데이터·오픈소스·상용 API의 **출처·라이선스·이용제약을 부록에 명시** (평가항목 3.5)
- **기술 제약 & 해결**
  - 환각 → 규칙엔진 병행 + 근거 인용 + 신뢰도 + 사람 검증
  - 민감정보 → 콘텐츠만 처리, PII 마스킹
  - 규정 최신성 → 규제 변경 추적 모듈

## 8. 사용자 시나리오

**주요 사용자**: ① 준법관리자(검토·승인) ② 콘텐츠 제작자(업로드·수정·게시)

```
1. 업로드     제작자가 베트남어 카드뉴스 제출
2. 자동 인식  OCR·번역 없이 원문에서 주장 추출
3. 심의·근거  위반 3건 + 규정 근거 + 수정안 도출
4. 검토       준법관리자가 승인·반려·코멘트 입력
5. 문서 출력  심의의견서 + 수정사항 정리본 생성
6. 게시       제작자가 수정 반영 후 적시 게시
```

> 핵심 분기점: ③ AI 판단과 ④ 사람 검토가 만나는 지점에서 **"책임은 사람, 속도는 AI"** 가 성립.

## 9. 기대 효과 & 확장성

- **정량**: 1차 심의 리드타임 주→분, 심의 리소스 비선형화, 외국인 미심의 리스크 최소화.
- **정성**: 마케팅 적시성 회복, 품질 편차·휴먼에러 감소, 준법관리자가 '리스크 의사결정'에 집중.
- **확장**: 언어 확장(태국어·캄보디아어), 계열사 확장(전북·광주은행·JB우리캐피탈), 영상·실시간, 규제 변경 자동 추적·재심의(능동형 Agent).

## 10. 평가 루브릭 대응 (예/본선, 각 20점 · 총 105점)

| 평가항목 | 배점 | 대응 |
|---|---|---|
| 1. 주제적합성·문제정의 | 20 | 실증 Pain Point(한투 경험) + JB 관점 중요성 |
| 2. 사업연계·고객가치 | 20 | 외국인 금융 1위 연계 + 내부통제·개인정보 인식(9번) |
| 3. AI Agent 설계 | 20 | 6단계 닫힌 루프(검증·사람검토) + 기술/출처 구체화 |
| 4. MVP 완성도 | 20 | 동작 데모(베트남어 카드뉴스 1건 완주) + 산출물 일관성 |
| 5. 혁신·확장·리스크 | 20 | 수작업→사전심의 전환 + 환각·책임소재·설명가능성 대응 |
| + 시연영상 | 5 | 데모 흐름 그대로 녹화 |

**주의**: 점수는 20×5 균등. 멀티모달·다국어를 화려하게 해도 그건 1·2번 일부일 뿐, **3번(Agent 설계)·5번(리스크)을 비우면 40점이 날아감**. → 검증/개선 루프와 리스크 대응을 반드시 명시.

## 11. MVP 스코프 (4일 + 작은 팀 기준)

| 구분 | 내용 |
|---|---|
| **반드시 코딩 (동작)** | ①~⑤ 한 사이클 + ⑥의 코멘트 취합·정리 출력. "베트남어 카드뉴스 1장 → 위반 3건 + 근거 + 수정안 → 사람 승인 → 심의의견서 출력"을 끝까지 완주 |
| **다이어그램·발표로** | 시스템 구성도, 영상/실시간 심의, 규제 자동 추적 |
| **본선 고도화로 이관** | 검토 의견 학습(재학습), 다국어 확장, 계열사 연계 |

> 데모 원칙: 기능 나열보다 **한 사이클을 완성도 있게**. 심사는 돌아가는 데모 하나를 더 좋아함.
